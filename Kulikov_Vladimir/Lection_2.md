## 1) Что такое программа

Определения (смысл одинаковый, формулировки из стандартов/ГОСТ):

- **Program (computer program)** — синтаксическая единица, которая соответствует правилам конкретного языка программирования и состоит из *объявлений* и *инструкций* для решения задачи.
- **Программа** — данные, предназначенные для управления компонентами системы обработки информации для реализации алгоритма.

Источники в лекции: ISO/IEC 2382:2015, ГОСТ 33707–2016, ГОСТ 19781–90.

---

## 2) Парадигмы программирования

### Императивное программирование
Характерные черты:
- в коде записываются **инструкции**;
- инструкции выполняются **последовательно**;
- последующие инструкции могут **читать** результаты предыдущих из памяти;
- инструкция может **записывать** данные в память.

**Идея:** программа описывает *как получить результат*.

Примеры: C, C++, Pascal, JavaScript, Java, Python и др.

### Декларативное программирование
- задаётся **спецификация решения**, т.е. **ожидаемый результат**, а не способ получения.

**Идея:** программа описывает *какой результат нужен*.

Примеры: HTML, LaTeX, SQL.

---

## 3) Базовые элементы программ

### Переменные
Хранят данные (значения в памяти).

### Инструкции (примеры из лекции)
- присвоение результата выражения: `mov AX, BX + 1`
- безусловные переходы: `JMP`, `GOTO`
- условные переходы (ветвления): `JNZ`

Инструкции объединяются в **блоки** — список последовательно выполняемых инструкций.

Из базовых элементов строятся сложные выражения: арифметика, логика, вызовы функций и т.п.

## 4) Управляющие конструкции в высокоуровневом коде

- **Ветвление**
  ```cpp
  if (A) { B } else { C }
````

* **Цикл**

  ```cpp
  while (A) { B }
  ```

* **Последовательное исполнение**

  ```cpp
  { A }; { B }
  ```

## 5) Теорема Бёма — Якопини (структурирование алгоритмов)

**Суть:** любой исполняемый алгоритм можно преобразовать в **структурированный** вид, где управление построено только на:

1. последовательности
2. ветвлении
3. цикле

Следствие: идея **отказа от `GOTO`** (исторически связано с работами Дейкстры).

## 6) Структурное программирование

Определение (смысл):

* метод построения программ с **иерархически вложенными конструкциями**, каждая из которых имеет **единственный вход и единственный выход**.

Источники в лекции: ISO/IEC 2382:2015, ГОСТ 33707–2016, ГОСТ 19781–90.

## 7) Блок-схема

**Блок-схема** — графическое представление задачи/алгоритма с помощью стандартных символов (операции, данные, поток, средства).

В лекции на слайде (страница с рисунком) показан пример простой блок-схемы.

Важно по ЕСПД:

* в стандартах ЕСПД есть термины **«схема программы»**, **«схема алгоритма»**
* термина **«блок-схема»** в ЕСПД нет (как стандартизованного названия)

## 8) Синтаксис C++: переменные и типы

### Определение переменной

* **Variable** — объект, заданный объявлением, с идентификатором, атрибутами данных, адресами и значениями.
* **Переменная** — языковой объект, который может принимать различные значения.

### Целочисленные типы (примеры объявлений)

```cpp
int a;
short int b;
long int c;
long long int d;

unsigned int e;
signed int f;

const unsigned long long int g;

char h;
signed char i;
unsigned char j;
```

### Прочие типы

```cpp
float a;
double b;
long double c;

bool f; // true / false

void;
```

### Пользовательские типы (typedef/using, struct)

```cpp
typedef int RUB;
RUB a;

using USD = int;
USD b;

struct Account {
  RUB money;
  float interest;
};

Account my_account;
// my_account.money = 10'000;
// my_account.interest = 14.5;
```

## 9) Циклы

### while + continue/break

```cpp
int k = 0;

while (k < 100) {
  k++;
  if (k == 10) continue;
  if (k == 40) break;
}
```

### for (и эквивалент через while)

```cpp
int k = 0;

for (k = 0; k < 100; ++k) {
  b++;
}

k = 0;
while (k < 100) {
  b++;
  k++;
}
```

### do...while

```cpp
int k = 0;

do {
  k++;
} while (k < 100);
```

## 11) Функции

### Процедура (void) — печатает сумму

```cpp
void print_sum(int a, int b)
{
  int c = a + b;
  printf("sum = %d", c);
}
```

### Функция (int) — возвращает сумму

```cpp
int sum(int a, int b)
{
  if (a == 0 and b == 0) return 0;

  int c = a + b;
  return c;
}

print_sum(1, 2);
int s = sum(1, 2);
```

## 12) Область видимости (scope)

* Область видимости переменной заканчивается ближайшими `{ }`, внутри которых она объявлена.
* При выходе из области видимости переменные уничтожаются.

Пример ошибки (переменная `b` объявлена внутри цикла и снаружи недоступна):

```cpp
int main()
{
  int a = 0;
  for (k = 0; k < 100; ++k) {
    int b = 0;
    a++; // OK
  }
  b = 10; // ERROR: b not exists!
}
```
