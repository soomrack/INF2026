## 1) Что такое «программа»?

**Program (computer program)** — syntactic unit that conforms to the rules of a particular programming language and that is composed of declarations and statements or instructions needed to solve a certain function, task, or problem.  
(ISO/IEC 2382:2015)

**Программа** — данные, предназначенные для управления конкретными компонентами системы обработки информации в целях реализации определенного алгоритма.  
(ГОСТ 33707–2016)

**Программа** — данные, предназначенные для управления конкретными компонентами системы обработки информации и реализации определенного алгоритма.  
(ГОСТ 19781–90)

## 2) Парадигмы программирования

### 2.1 Императивное программирование
**Императивное программирование** — парадигма, для которой характерно:

- в исходном коде записываются **инструкции**;
- инструкции выполняются **последовательно**;
- данные из памяти, полученные ранее, могут **читаться** последующими инструкциями;
- результаты выполнения инструкции могут **записываться в память**.

> Идея: программа состоит из инструкций **как получить результат**.

**Примеры:** C, C++, Pascal, JavaScript, Java, Python…

### 2.2 Декларативное программирование
**Декларативное программирование** — парадигма, в которой задаётся **спецификация решения**, то есть описывается **ожидаемый результат**, а не способ его получения.

> Идея: программа состоит из описания **ожидаемого результата**.

**Примеры:** HTML, LaTeX, SQL.

---

## 3) Базовые элементы программ

Базовые элементы программы:
- **переменные**;
- **инструкции**, например:
  - присвоение результата выражения: `mov AX, BX + 1`
  - безусловные переходы: `JMP`, `GOTO`
  - условные переходы (ветвления): `JNZ`

Дополнительно:
- инструкции можно объединять в **блоки** (список последовательно выполняемых инструкций);
- из базовых элементов строятся сложные выражения: арифметические/логические операции, вычисления функций и т.п.

## 4) Высокоуровневый код: базовые конструкции управления

### Ветвление
```cpp

````

### Цикл

```cpp

```

### Последовательное исполнение

```cpp

```

## 5) Теорема Бёма — Якопини

**Теорема Бёма — Якопини (1966):** любой исполняемый алгоритм может быть преобразован к **структурированному** виду, где ход выполнения определяется только тремя типами управления:

1. последовательное исполнение
2. ветвление
3. цикл

Следствие (идея): **отказ от оператора `GOTO`** (в т.ч. в духе работы Дейкстры “Go To Statement Considered Harmful”, 1968).

## 6) Структурное программирование

Определения (по смыслу):

* **Structured programming** — method for constructing programs using only hierarchically arranged constructs each having a single entry point and a single exit point.
  (ISO/IEC 2382:2015)

* **Структурное программирование** — всякая техника разработки ПО, использующая структурное проектирование, результатом которой является разработка структурированных программ.
  (ГОСТ 33707–2016)

* **Структурное программирование** — метод построения программ, использующий только иерархически вложенные конструкции, каждая из которых имеет единственную точку входа и единственную точку выхода.
  (ГОСТ 19781–90)

## 7) Блок-схема

На слайде приведён пример **блок-схемы программы** (страница с рисунком в лекции).

**Блок-схема** — графическое представление задачи для анализа или решения с помощью специальных символов, обозначающих такие элементы как операции, данные, поток или технические средства.
(ГОСТ 33707–2016)

**Block diagram** — diagram of a system in which the principal parts or functions are represented by blocks connected by lines that show the relationships of the blocks.
(ISO/IEC 2382:2015)

## 8) ЕСПД

**ЕСПД (Единая система программной документации)** — комплекс государственных стандартов, устанавливающий взаимоувязанные правила разработки, оформления и обращения программ и программной документации.

Замечания из лекции:

* есть термины: **схема программы**, **схема алгоритма**;
* нет термина: **блок-схема** (как стандартизованного названия в ЕСПД).

## 9) Синтаксис C++: переменные

**Variable** — quadruple, established by a declaration or an implicit declaration, that consists of an identifier, a set of data attributes, one or more addresses, and data values, where the relationship between the addresses and the data values may vary.
(ISO/IEC 2382:2015)

**Переменная** — языковой объект, который может принимать различные значения.
(ГОСТ 33707–2016)

## 11) Циклы

### 11.1 while + continue/break

```cpp
int k = 0;

while (k < 100) {
  k++;
  if (k == 10) continue;
  if (k == 40) break;
}
```

### 11.2 for (и эквивалент через while)

```cpp
int k = 0;

for (k = 0; k < 100; ++k) {
  b++;
}

k = 0;
while (k < 100) {
  b++;
  k++;
}
```

### 11.3 do...while

```cpp
int k = 0;

do {
  k++;
} while (k < 100);
```

## 13) Функции

### 13.1 Процедура (void) — печатает сумму

```cpp
void print_sum(int a, int b)
{
  int c = a + b;
  printf("sum = %d", c);
}
```

### 13.2 Функция (int) — возвращает сумму

```cpp
int sum(int a, int b)
{
  if (a == 0 and b == 0) return 0;

  int c = a + b;
  return c;
}

print_sum(1, 2);
int s = sum(1, 2);
```

## 14) Область видимости (scope)

* Область видимости переменных заканчивается ближайшими `{ }`, внутри которых они объявлены.
* Переменные уничтожаются при выходе из области видимости.

Пример (переменная `b` объявлена внутри цикла, поэтому снаружи недоступна):

```cpp
int main()
{
  int a = 0;

  for (k = 0; k < 100; ++k) {
    int b = 0;
    a++; // OK
  }

  b = 10; // ERROR: b not exists
}
```
