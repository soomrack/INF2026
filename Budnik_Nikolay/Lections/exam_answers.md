# Exam answers

## 1. Introduction and development tools

### 1. Что такое информатика?
Информатика — это наука об информации: о ее создании, хранении, обработке, передаче и использовании. В учебном курсе информатика рассматривается как область, связанная с программированием, устройством компьютера, операционными системами, сетями и базами данных


### 2. Достоинства и недостатки выбора языка С для обучения
С удобен для обучения тем, что он близок к устройству компьютера, показывает работу памяти, указателей, типов данных и компиляции. На нем пишут системное ПО, драйверы и код для микроконтроллеров. Недостатки C — высокая ответственность программиста, ручная работа с памятью, слабая защита от ошибок и отсутствие многих удобных механизмов высокого уровня


### 3. Достоинства и недостатки выбора языка С++ для обучения
C++ полезен для обучения, потому что сохраняет связь с C и низкоуровневым программированием, но дополнительно дает ООП, классы, конструкторы, деструкторы, RAII и стандартную библиотеку. Язык имеет стандарт, много компиляторов и применяется в системном ПО, микроконтроллерах, играх и инженерных задачах. Недостаток C++ — сложный синтаксис и большое количество возможностей, из-за чего новичку легко запутаться


### 4. Зачем нужен стандарт для языка?
Стандарт языка нужен, чтобы разные компиляторы одинаково понимали синтаксис и поведение программ. Он фиксирует правила языка, типы данных, основные конструкции и требования к реализации. Благодаря стандарту код можно переносить между системами и компиляторами с меньшим количеством изменений


### 5. Интегрированная программная среда и интегрированная среда разработки
Интегрированная программная среда — это набор программных и аппаратных средств для разработки программ, объединенный общим интерфейсом. Интегрированная среда разработки, или IDE, обычно включает редактор кода, инструменты сборки, запуск программы, отладчик и средства настройки проекта. Примеры: Visual Studio, VS Code, Arduino IDE, Vim, Emacs


### 6. Что такое компилятор?
Компилятор — это программа, которая переводит исходный код на языке программирования в машинный код или другой исполняемый формат. Например, компилятор C++ берет файл `.cpp`, проверяет синтаксис и создает программу, которую можно запустить. Примеры компиляторов: g++, clang, MSVC, MinGW


### 7. Детальное объяснение программы "Hello World"
Программа `Hello World` обычно нужна для проверки, что среда разработки и компилятор настроены правильно. В C/C++ строка `#include <stdio.h>` подключает библиотеку для функции `printf`. Функция `main()` является точкой входа в программу: выполнение начинается именно с нее. Команда `printf("Hello World!");` выводит текст в терминал


### 8. Что такое терминал?
Терминал — это средство взаимодействия пользователя с системой через текстовые команды. Через терминал можно запускать программы, компилировать код, просматривать файлы и работать с инструментами разработки. В контексте курса терминал нужен, например, чтобы выполнить команды компиляции и запуска программы


### 9. Распределенная система хранения кода
Распределенная система хранения кода позволяет хранить историю изменений проекта не только на одном сервере, но и у каждого разработчика локально. Пример — Git. Он позволяет смотреть изменения по коммитам, возвращаться к старым версиям и работать совместно. GitHub — платформа для хранения репозиториев и совместной разработки на основе Git


## 2. Programming paradigms and program structure

### 10. Что такое программа?
Программа — это набор данных, инструкций, объявлений и команд, которые управляют работой компьютера для выполнения определенной задачи. Она записывается по правилам конкретного языка программирования и реализует некоторый алгоритм


### 11. Императивное программирование
Императивное программирование — это подход, при котором программист описывает последовательность действий, которые должен выполнить компьютер. Программа состоит из инструкций: присваиваний, условий, циклов, переходов и операций с памятью. Примеры языков: C, C++, Pascal, Java, JavaScript, Python


### 12. Декларативное программирование
Декларативное программирование — это подход, при котором программист описывает не последовательность действий, а ожидаемый результат. То есть важнее указать, что нужно получить, а не как именно это вычислять шаг за шагом. Примеры: HTML, LaTeX, SQL


### 13. Из чего состоит программа?
Программа состоит из переменных, инструкций и блоков инструкций. Основные элементы — присваивание, арифметические и логические выражения, ветвления, циклы и функции. Более сложные программы строятся из этих базовых конструкций


### 14. Теорема Бема-Якопини
Теорема Бема-Якопини утверждает, что любой исполняемый алгоритм можно представить в структурированном виде с помощью трех управляющих конструкций: последовательного выполнения, ветвления и цикла. Из этого следует, что для построения алгоритмов не обязательно использовать неструктурированные переходы вроде `GOTO`


### 15. Структурное программирование
Структурное программирование — это метод построения программ с использованием иерархически организованных конструкций, у которых есть один вход и один выход. Основная идея — писать программу так, чтобы ее выполнение было понятно по структуре блоков, условий, циклов и функций


### 16. Что такое блок-схема?
Блок-схема — это графическое представление алгоритма или программы с помощью специальных символов. В ней показываются действия, условия, ввод-вывод и направление выполнения. Формально в ЕСПД чаще используются термины «схема алгоритма» и «схема программы»


### 17. Что такое ЕСПД?
ЕСПД — это Единая система программной документации. Она представляет собой комплекс стандартов, которые задают правила разработки, оформления и обращения программ и программной документации. ЕСПД нужна для единообразного оформления программных документов


## 3. Basic C++ syntax

### 18. Что такое переменная?
Переменная — это именованный объект в программе, который хранит значение определенного типа. У переменной есть имя, тип, область видимости, адрес в памяти и текущее значение. Значение переменной может изменяться во время выполнения программы, если она не объявлена как константа


### 19. Базовые типы данных
К базовым типам данных в C++ относятся целочисленные типы `int`, `short`, `long`, `long long`, символьный тип `char`, логический тип `bool`, вещественные типы `float`, `double`, `long double`, а также `void`. Для целых типов могут использоваться модификаторы `signed` и `unsigned`


### 20. Пользовательские типы данных
Пользовательские типы данных создаются самим программистом. В C++ для этого можно использовать `typedef`, `using`, `struct`, `class` и другие механизмы. Например, можно создать тип `USD` на основе `int` или структуру `Account`, в которой хранятся деньги и процентная ставка


### 21. Циклы
Циклы нужны для многократного выполнения блока кода. В C++ есть цикл `while`, который выполняется, пока условие истинно, цикл `for`, удобный для счетчиков, и цикл `do while`, который выполняет тело хотя бы один раз. Внутри циклов можно использовать `break` для выхода и `continue` для перехода к следующей итерации


### 22. Условные операторы
Условные операторы позволяют выполнять разные участки кода в зависимости от условия. Основная конструкция — `if`, а вместе с ней может использоваться `else`. Если условие истинно, выполняется один блок, если ложно — другой


### 23. Функции
Функция — это именованный блок кода, который выполняет отдельную задачу. У функции может быть тип возвращаемого значения, имя, параметры и тело. Например, функция может напечатать сумму двух чисел или вернуть результат сложения через `return`


### 24. Область видимости
Область видимости определяет, где в программе можно обращаться к переменной. В C++ область видимости обычно ограничивается ближайшими фигурными скобками, внутри которых переменная объявлена. После выхода из этой области локальная переменная уничтожается и использовать ее уже нельзя


## 4. Memory and data representation

### 25. Что такое бит и что такое байт?
Бит — минимальная единица информации, которая может принимать одно из двух значений: 0 или 1. Байт — группа битов, обрабатываемая как единое целое. В современном понимании байт обычно равен 8 битам, но для точного обозначения 8-битного байта также используют термин «октет»


### 26. Единицы измерения объема памяти
Объем памяти измеряется в битах и байтах. Для десятичных кратных единиц используют приставки СИ: килобайт, мегабайт, гигабайт, терабайт и так далее, где множитель основан на степенях 10. Для двоичных кратных единиц используют приставки IEC: кибибайт, мебибайт, гибибайт, тебибайт, где множитель основан на степенях 1024


### 27. Позиционная система счисления, двоичная, восьмеричная, шестнадцатеричная
Позиционная система счисления — это система, где значение цифры зависит от ее позиции в числе. В десятичной системе основание равно 10, в двоичной — 2, в восьмеричной — 8, в шестнадцатеричной — 16. В C++ десятичное число пишется как `13`, шестнадцатеричное как `0x0D`, восьмеричное как `015`, двоичное как `0b00001101`


### 28. Порядок байтов
Порядок байтов показывает, в каком порядке байты многобайтового числа размещаются в памяти. `big-endian` хранит сначала старший байт, `little-endian` — сначала младший байт. В сетевых протоколах часто используется big-endian, а в архитектуре x86 — little-endian. ARM может поддерживать переключаемый порядок байтов


### 29. Адрес переменной, ссылка, указатель
Адрес переменной — это место в памяти, где хранится ее значение. Указатель хранит адрес другой переменной, например `int* a`. Оператор `&` берет адрес, а `*` разыменовывает указатель, то есть дает доступ к значению по адресу. Ссылка `int& e = c` является псевдонимом уже существующей переменной


## 5. Strings, encodings and algorithms

### 30. Что такое cтрока?
Строка — это последовательность элементов одного типа, чаще всего символов, которая рассматривается как единое целое. В программировании строки применяются для хранения и обработки текста: слов, сообщений, команд, имен файлов и других текстовых данных


### 31. Представление символов в компьютере. Кодировки
Компьютер хранит символы в виде чисел, а кодировка задает соответствие между символами и числовыми кодами. Примеры кодировок: ASCII, CP1251, KOI8-R, CP866, Unicode. Разные кодировки могут по-разному представлять одни и те же символы, особенно если речь идет не о латинице, а о кириллице или других письменностях


### 32. Строки завершающиеся нулем
Строка, завершающаяся нулем, хранится как непрерывная последовательность байтов, в конце которой находится байт `\0`. Такой формат используется в C. Например, строка `"Hello"` занимает 6 байт: пять символов и завершающий нулевой байт. Если убрать нулевой байт, вывод может продолжиться по случайным данным в памяти


### 33. Юникод
Unicode — это стандарт кодирования символов, созданный для поддержки текстов на разных языках и письменностях. Он задает универсальные кодовые точки для символов. Для хранения Unicode используются разные форматы, например UTF-8, UTF-16 и UTF-32. UTF-8 особенно распространен в интернете


### 34. Равномерное кодирование
Равномерное кодирование — это способ представления данных, при котором каждый символ исходного алфавита кодируется словом одинаковой длины. Такой подход проще для обработки, потому что границы символов легко определить, но он может расходовать память неэффективно


### 35. Префиксный код
Префиксный код — это код переменной длины, в котором никакое кодовое слово не является началом другого кодового слова. Это свойство называется условием Фано. Благодаря ему последовательность кодовых слов можно однозначно декодировать без специальных разделителей


### 36. Преимущества std::string
`std::string` удобнее C-строк, потому что сам хранит размер строки и управляет памятью. С ним проще добавлять символы, изменять длину, вставлять и заменять подстроки. Он уменьшает риск ошибок с завершающим нулем и делает работу со строками более безопасной и читаемой


### 37. Алгоритмы на строках
К основным алгоритмам на строках относятся сравнение строк, сортировка, поиск подстроки, поиск слов из словаря, оценка близости строк и вычисление расстояния между строками. Такие алгоритмы используются в поиске, автодополнении, проверке орфографии, анализе текста и обработке данных


### 38. Регулярные выражения
Регулярное выражение, или regex, — это формальный язык для поиска и изменения фрагментов текста по шаблону. С помощью регулярных выражений можно находить цифры, слова, специальные символы и заменять найденные участки. Например, шаблон `[0-9]+` означает одну или несколько цифр подряд


### 39. Сложность алгоритмов
Сложность алгоритма — это оценка ресурсов, необходимых для его выполнения в зависимости от размера входных данных. Временная сложность показывает, как растет время работы, а пространственная — как растет потребление памяти. Часто используют обозначения `O(1)`, `O(log n)`, `O(n)`, `O(n²)`, `O(2ⁿ)`


## 6. Object-oriented programming

### 40. Что такое объектно-ориентированное программирование?
Объектно-ориентированное программирование — это подход, в котором программа строится вокруг объектов. Объект объединяет данные и операции над ними. Главные идеи ООП — инкапсуляция, наследование и полиморфизм. ООП помогает скрывать сложность, уменьшать дублирование кода и расширять программу без постоянного переписывания старых частей


### 41. Что такое класс и объект?
Класс — это шаблон или описание для создания объектов. Он задает поля, методы и правила работы с данными. Объект — это конкретный экземпляр класса, который существует во время выполнения программы и имеет собственное состояние. Например, класс `Thermometer` описывает термометр, а `top` и `bottom` могут быть двумя разными объектами этого класса


### 42. Наследование
Наследование — это механизм, при котором новый класс получает поля и методы базового класса. Оно позволяет расширять существующий код без копирования одинаковых частей. Например, `ThermometerTested` может наследоваться от `Thermometer` и дополнительно иметь поле `is_broken` и метод самопроверки


### 43. Полиморфизм (два вида)
Полиморфизм означает, что разные объекты могут по-разному реагировать на одинаковое обращение. Первый вид — полиморфизм в параметрах, когда выбор перегруженной функции определяется при компиляции. Второй вид — полиморфизм в методах, когда через `virtual` и `override` выбор метода происходит во время выполнения программы, это называется поздним связыванием


### 44. Инкапсуляция
Инкапсуляция — это объединение данных и методов, работающих с этими данными, внутри одного класса. Она скрывает детали реализации от внешнего кода. Благодаря инкапсуляции объект можно использовать через понятные методы, не вмешиваясь напрямую в его внутреннее устройство


### 45. Контроль доступа к полям объекта (public, private)
Контроль доступа определяет, какие поля и методы класса доступны снаружи. `public` означает открытый доступ, а `private` — доступ только внутри класса. Обычно данные делают `private`, а для работы с ними создают `public` методы, чтобы защитить объект от некорректных изменений


### 46. Open-Closed принцип
Open-Closed principle означает, что программные сущности должны быть открыты для расширения, но закрыты для изменения. Проще говоря, если нужно добавить новое поведение, лучше создать наследника или расширение, а не переписывать уже работающий базовый класс. Это снижает риск сломать старый код


### 47. Конструкторы и деструкторы
Конструктор — это специальный метод класса, который вызывается при создании объекта и подготавливает его начальное состояние. Деструктор вызывается при уничтожении объекта и нужен для освобождения ресурсов или завершения работы. В C++ конструкторы и деструкторы позволяют связывать жизненный цикл объекта с управлением ресурсами


### 48. Принцип RAII
RAII означает Resource Acquisition Is Initialization. Смысл принципа в том, что ресурс захватывается при создании объекта и освобождается при его уничтожении. Например, память можно выделить в конструкторе, а освободить в деструкторе. Это помогает избежать утечек памяти и делает управление ресурсами надежнее


## 7. Operating systems and Linux

### 49. Что такое операционная система? Примеры операционных систем
Операционная система — это программная среда, которая управляет аппаратными ресурсами компьютера и предоставляет программам удобный интерфейс для работы. Она управляет процессами, памятью, файлами, устройствами ввода-вывода и правами доступа. Примеры операционных систем: Linux, Windows, macOS, Android, iOS


### 50. Что такое POSIX?
POSIX — это набор стандартов, описывающих интерфейсы операционных систем, совместимых с Unix-подходом. Он нужен, чтобы программы могли переноситься между разными системами с похожим API. POSIX описывает работу процессов, файлов, потоков, сигналов, прав доступа и других системных механизмов


### 51. Что такое процесс?
Процесс — это выполняющийся экземпляр программы. У процесса есть собственное адресное пространство, состояние, идентификатор, открытые файлы и выделенные ресурсы. Операционная система планирует выполнение процессов и переключает процессор между ними


### 52. Что такое файл?
Файл — это именованный объект для хранения данных во внешней памяти или абстракция доступа к ресурсу. В Unix/Linux многие объекты представлены как файлы: обычные файлы, каталоги, устройства, каналы и сокеты. Такой подход упрощает взаимодействие программ с разными ресурсами


### 53. Чем отличается пространство ядра (kernel space) и пространство пользователя (user space)?
Пространство ядра — это защищенная область, где выполняется код ядра операционной системы и драйверов. Пространство пользователя — область, где работают обычные программы. Пользовательские программы не могут напрямую обращаться к аппаратуре и критическим структурам ядра, поэтому используют системные вызовы


### 54. Чем отличаются дистрибутивы Linux между собой? Какой дистрибутив разрабатывается в России?
Дистрибутивы Linux отличаются набором программ, системой управления пакетами, настройками, графической оболочкой, политикой обновлений и целевой аудиторией. Например, одни удобны для серверов, другие — для рабочего стола или встраиваемых систем. В России разрабатывается дистрибутив ALT Linux


### 55. Преимущества распространения программного обеспечения по модели opensource
Open source дает доступ к исходному коду, поэтому программу можно изучать, проверять, изменять и адаптировать под свои задачи. Такая модель повышает прозрачность, облегчает совместную разработку и позволяет находить ошибки силами сообщества. Также она снижает зависимость от одного поставщика


### 56. Какие модели управления доступа к ресурасам используются в Linux? (дискретная модель, расширенные списки доступа, мандатная модель)
В Linux используется дискретная модель доступа: права назначаются владельцу, группе и остальным пользователям через чтение, запись и выполнение. Также могут применяться расширенные списки доступа, ACL, которые позволяют задавать права более гибко для отдельных пользователей и групп. Мандатная модель реализуется через механизмы вроде SELinux или AppArmor, где доступ определяется системной политикой безопасности


### 57. Что такое системы реального времени?
Система реального времени — это система, для которой важен не только правильный результат, но и время его получения. Такая система должна реагировать на события в заданные временные ограничения. Примеры: управление промышленным оборудованием, автомобильная электроника, робототехника, медицинские устройства


## 8. Networks and industrial data

### 58. Что такое промышленная сеть передачи данных?
Промышленная сеть передачи данных — это сеть, предназначенная для обмена информацией между промышленными устройствами: датчиками, контроллерами, приводами, роботами, станками и системами управления. Для таких сетей важны надежность, предсказуемая задержка, устойчивость к помехам и работа в тяжелых условиях


### 59. Какие существуют типы линий связи? В чем их различие и преимущества
Основные типы линий связи — медные проводные линии, оптоволоконные линии и беспроводные каналы. Медные линии дешевле и проще в подключении, но чувствительнее к помехам и имеют ограничения по расстоянию. Оптоволокно обеспечивает высокую скорость, большую дальность и устойчивость к электромагнитным помехам. Беспроводная связь удобна там, где сложно прокладывать кабель, но обычно менее предсказуема по задержкам и надежности


### 60. Эталонная модель OSI
Модель OSI — это семиуровневая модель описания сетевого взаимодействия. Уровни: физический, канальный, сетевой, транспортный, сеансовый, представления и прикладной. Каждый уровень решает свою задачу и передает данные соседним уровням. Модель помогает понимать, где именно возникает проблема в сети и какие протоколы за что отвечают


### 61. Стандарт 802.3 (Ethernet)
IEEE 802.3 — это группа стандартов, описывающих Ethernet. Ethernet определяет работу проводных локальных сетей на физическом и канальном уровнях, включая формат кадров, доступ к среде передачи и варианты скоростей. Сегодня Ethernet широко используется в офисных, домашних и промышленных сетях


### 62. Промышленный Ethernet
Промышленный Ethernet — это применение Ethernet-технологий в промышленной автоматизации. Он отличается повышенными требованиями к надежности, задержкам, синхронизации и устойчивости оборудования. Примеры промышленных Ethernet-протоколов: PROFINET, EtherCAT, EtherNet/IP, Modbus TCP


### 63. Какие данные создаются при промышленном производстве?
При промышленном производстве создаются данные датчиков, параметры технологического процесса, команды управления, состояние оборудования, аварии, журналы событий, данные контроля качества, данные о продукции, энергии и обслуживании. Эти данные используются для мониторинга, диагностики, оптимизации производства и предиктивного обслуживания


### 64. Что такое большие данные? Шесть "V"
Большие данные — это наборы данных, которые трудно хранить, обрабатывать и анализировать обычными методами из-за их масштаба и сложности. Шесть «V» обычно описывают их свойства: Volume — объем, Velocity — скорость поступления, Variety — разнообразие, Veracity — достоверность, Value — ценность, Variability — изменчивость


### 65. Проблемы обработки больших данных
Основные проблемы больших данных — хранение огромных объемов информации, высокая скорость поступления, неоднородные форматы, шум и ошибки в данных, сложность очистки, необходимость масштабируемых вычислений и защита информации. Кроме того, важно не просто собрать данные, а извлечь из них полезную ценность для принятия решений
