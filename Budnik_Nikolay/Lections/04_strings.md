# Лекция 04. Строки, кодировки и сложность алгоритмов

## Кратко о лекции

Лекция объясняет, как символы и строки хранятся в компьютере, зачем нужны кодировки, чем удобен `std::string`, что такое регулярные выражения и как оценивают сложность алгоритмов

## 1. Символ и строка

Символ — элемент набора данных, который используется для представления, организации или управления информацией

Строка — последовательность однотипных элементов, чаще всего символов

Примеры:

```cpp
char letter = 'A';
```

```cpp
"Hello"
```

Символ — один элемент, строка — набор элементов

## 2. ASCII

ASCII — одна из базовых таблиц кодирования символов

Она задаёт соответствие между числом и символом

Например, в ASCII кодируются:

- латинские буквы
- цифры
- знаки препинания
- управляющие символы

ASCII важна исторически, потому что многие современные кодировки совместимы с ней в первой части таблицы

## 3. Кодировки для кириллицы

Для русского текста раньше использовались разные кодировки:

- CP1251
- KOI8-R
- CP866

Проблема старых кодировок в том, что один и тот же байт мог означать разные символы в разных таблицах

Из-за этого появлялись «кракозябры», когда текст открывался не в той кодировке

## 4. C-строки с нулевым окончанием

Null-terminated string — строка, которая хранится как последовательность байтов и заканчивается нулевым байтом `\0`

Пример:

```cpp
char str[] = "Hello";
printf("%s", str);
```

Важно: строка `"Hello"` занимает 6 байт, а не 5

Причина:

- `H`
- `e`
- `l`
- `l`
- `o`
- `\0`

Нулевой байт показывает, где строка заканчивается

Если заменить `\0` на обычный символ, функции вывода могут читать память дальше и выводить мусор

## 5. Unicode

Unicode — стандарт, созданный для представления символов разных письменностей мира

Он нужен, чтобы можно было работать не только с английским текстом, но и с русским, китайским, арабским, математическими символами и другими знаками

Основные варианты кодирования Unicode:

- UTF-8
- UTF-16
- UTF-32

Unicode задаёт символы, а UTF-8, UTF-16 и UTF-32 задают способ хранения этих символов в памяти

## 6. UTF-8

UTF-8 — кодировка Unicode с переменной длиной символа

Один символ может занимать:

- 1 байт
- 2 байта
- 3 байта
- 4 байта

Плюсы UTF-8:

- совместимость с ASCII
- экономное хранение английского текста
- поддержка большого числа письменностей
- широкое использование в интернете

Для программирования важно помнить: длина строки в байтах не всегда равна количеству видимых символов

## 7. Кодирование

Равномерное кодирование — метод, где каждый символ кодируется словом одинаковой длины

Префиксный код — код переменной длины, в котором ни одно кодовое слово не является началом другого

Условие Фано: никакое кодовое слово не может быть началом другого кодового слова

Это нужно, чтобы последовательность кодов можно было однозначно декодировать

## 8. std::string

`std::string` — стандартный класс C++ для работы со строками

Пример:

```cpp
#include <string>
#include <iostream>

std::string str;
str = "Hello World";
str.push_back('!');
std::cout << str << std::endl;
```

`std::string` удобнее C-строк, потому что сам хранит размер и предоставляет готовые методы

## 9. Полезные методы std::string

Частые операции:

```cpp
str.length();
str.size();
str.insert(2, "text");
str.replace(2, 3, "abc");
str.resize(10);
```

Важно: `length()` и `size()` обычно возвращают длину в байтах, а не количество Unicode-символов

## 10. Алгоритмы на строках

Основные задачи со строками:

- сравнение строк
- оценка близости строк
- сортировка строк
- поиск подстроки
- поиск слова из словаря в тексте

Такие задачи встречаются в поиске, редакторах, компиляторах, базах данных и обработке естественного языка

## 11. Регулярные выражения

Регулярное выражение — формальный шаблон для поиска и изменения текста

Пример идеи:

```text
a1234567b -> aDIGITSb
```

Здесь последовательность цифр заменяется словом `DIGITS`

Регулярные выражения удобны для:

- поиска шаблонов
- проверки формата строки
- замены фрагментов текста
- обработки логов и таблиц

## 12. Сложность алгоритмов

Сложность алгоритма показывает, сколько ресурсов нужно алгоритму при росте входных данных

Два основных вида сложности:

- временная сложность
- пространственная сложность

Временная сложность оценивает количество операций

Пространственная сложность оценивает расход памяти

## 13. Основные классы сложности

Типичные оценки:

| Обозначение | Смысл |
|---|---|
| O(1) | константная сложность |
| O(log n) | логарифмическая сложность |
| O(n) | линейная сложность |
| O(n²) | квадратичная сложность |
| O(p(n)) | полиномиальная сложность |
| O(2ⁿ) | экспоненциальная сложность |

Чем быстрее растёт сложность, тем хуже алгоритм масштабируется на больших данных

## 14. P и NP

Проблема `P ? NP` связана с вопросом: можно ли быстро решать задачи, для которых готовое решение можно быстро проверить

Это одна из известных проблем современной теории алгоритмов

## Итог лекции

Строки в памяти зависят от кодировки, поэтому важно различать символы, байты и видимую длину текста. Для удобной работы в C++ используют `std::string`, а эффективность строковых алгоритмов оценивают через сложность
