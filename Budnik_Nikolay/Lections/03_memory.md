# Лекция 03. Память, биты, байты и указатели

## Кратко о лекции

Лекция посвящена тому, как информация хранится в памяти: что такое бит, байт, октет, системы счисления, порядок байтов, адреса переменных и указатели

## 1. Бит

Бит — минимальная единица информации

Он может принимать только два значения:

- `0`
- `1`

В двоичной системе все данные компьютера в итоге представлены через нули и единицы

## 2. Байт

Байт — это группа битов, которая обрабатывается как единое целое

В современных системах обычно под байтом понимают 8 бит, но в стандартах C и C++ важно, что байт — это минимальная адресуемая единица памяти

## 3. Бит и байт в C

В языке C:

- бит хранит одно из двух значений
- отдельный бит не всегда имеет собственный адрес
- байт является адресуемой единицей памяти
- байт должен быть достаточен для хранения символа базового набора

Важно: программа обычно работает не с отдельными битами, а с байтами и объектами в памяти

## 4. Байт в C++

В модели памяти C++ байт — фундаментальная единица хранения

Свойства байта:

- состоит из последовательности битов
- имеет уникальный адрес
- является частью непрерывной памяти программы
- достаточно велик для хранения символов базовой кодировки

Память программы можно представить как длинную последовательность байтов

## 5. Октет

Октет — это ровно 8 бит

Отличие:

- байт зависит от модели языка и системы
- октет всегда равен 8 битам

На практике байт почти всегда равен октету, но в стандартах эти термины разделяют

## 6. Приставки СИ

Десятичные приставки основаны на степенях 10

| Значение | Приставка | Пример |
|---:|---|---|
| 10³ | кило | кБ |
| 10⁶ | мега | МБ |
| 10⁹ | гига | ГБ |
| 10¹² | тера | ТБ |
| 10¹⁵ | пета | ПБ |
| 10¹⁸ | экса | ЭБ |
| 10²¹ | зетта | ЗБ |
| 10²⁴ | йотта | ИБ |

Эти приставки удобны для дисков, сетей и общих объёмов данных

## 7. Приставки МЭК / IEC

Двоичные приставки основаны на степенях 2

| Значение | Приставка | Пример |
|---:|---|---|
| 2¹⁰ | киби | КиБ |
| 2²⁰ | меби | МиБ |
| 2³⁰ | гиби | ГиБ |
| 2⁴⁰ | теби | ТиБ |
| 2⁵⁰ | пеби | ПиБ |
| 2⁶⁰ | эксби | ЭиБ |

Главное отличие:

- `1 кБ = 1000 байт`
- `1 КиБ = 1024 байта`

## 8. Системы счисления

Позиционная система счисления — система, где значение цифры зависит от её позиции

Основные системы:

- десятичная: `0–9`
- двоичная: `0–1`
- восьмеричная: `0–7`
- шестнадцатеричная: `0–9` и `A–F`

Компьютеру ближе двоичная система, но человеку удобнее десятичная и шестнадцатеричная

## 9. Числа в C++

Одно и то же число можно записать разными способами

```cpp
int a = 13;          // десятичная запись
int b = 0x0D;        // шестнадцатеричная запись
int c = 015;         // восьмеричная запись
int d = 0b00001101;  // двоичная запись
```

Сдвиги работают с битовым представлением числа

```cpp
d >> 2;
d << 1;
```

`>>` сдвигает биты вправо, `<<` сдвигает биты влево

## 10. Порядок байтов

Порядок байтов показывает, как многобайтовое число лежит в памяти

Основные варианты:

- **big-endian** — сначала старший байт
- **little-endian** — сначала младший байт
- **bi-endian** — порядок можно переключать
- **middle-endian** — смешанный порядок

Примеры:

- TCP/IP использует big-endian
- x86 и USB обычно используют little-endian
- ARM может работать как bi-endian

## 11. Адрес переменной

У каждой переменной в памяти есть адрес

Указатель хранит адрес другой переменной

```cpp
int c = 0;
int* a;

a = &c;
*a = 3;
```

Здесь:

- `&c` берёт адрес переменной `c`
- `a` хранит этот адрес
- `*a` обращается к значению по адресу

После `*a = 3` переменная `c` тоже станет равна `3`

## 12. Ссылки

Ссылка — это псевдоним уже существующей переменной

```cpp
int c = 10;
int& e = c;
```

После этого `e` и `c` обозначают одно и то же значение

## 13. Нулевой указатель

Если указатель никуда не должен указывать, используют `nullptr`

```cpp
int* a = nullptr;
```

В C часто встречается `NULL`, но в современном C++ предпочтительнее `nullptr`

## 14. Массивы и арифметика указателей

Массив хранит элементы подряд в памяти

```cpp
int a[10];

a[0] = 10;
a[9] = 19;
```

Имя массива указывает на его первый элемент

```cpp
int* b = a;
b += 1;
```

После `b += 1` указатель переходит к следующему элементу массива, а не просто к следующему байту

Запись `b[2]` эквивалентна `*(b + 2)`

## Итог лекции

Память программы состоит из байтов, у каждого байта есть адрес. Указатели позволяют работать с адресами напрямую, поэтому они дают большую гибкость, но требуют аккуратности
