# Блок 4. Memory and data representation

## Вопрос 25. Что такое бит и что такое байт?

Бит и байт — это базовые единицы представления данных в компьютере. Бит — минимальная единица информации, которая может принимать одно из двух значений: `0` или `1`. Все данные в компьютере в конечном итоге представляются через последовательности битов: числа, символы, изображения, звук, сетевые пакеты и команды программы.

Байт — это группа битов, которая обрабатывается как единое целое и обычно является минимальной адресуемой единицей памяти. В современных системах байт почти всегда равен 8 битам, но технически для точного обозначения 8 бит используют термин «октет». В стандартах C и C++ важно помнить: байт — адресуемая единица памяти, а октет — ровно 8 бит.

Смысл бита зависит от договорённости о кодировании. Одна и та же последовательность битов может быть целым числом, символом, частью изображения или машинной инструкцией. Поэтому компьютер хранит биты, а программа и формат данных определяют, как эти биты интерпретировать.

Пример: если переменная занимает 4 байта, то при обычном восьмибитном байте это `4 · 8 = 32` бита. Поэтому тип `int` на многих платформах занимает 32 бита, хотя точный размер зависит от компилятора и системы. Типичная ошибка — думать, что бит и байт одно и то же. Коротко: бит хранит один `0` или `1`, а байт обычно состоит из 8 битов и служит минимальной удобной единицей памяти.

## Вопрос 26. Единицы измерения объёма памяти

Единицы измерения объёма памяти показывают, сколько битов или байтов требуется для хранения данных. Базовая практическая единица — байт, а для больших объёмов используют кратные единицы. Десятичные приставки СИ основаны на степенях 10: килобайт — 1000 байт, мегабайт — 10⁶ байт, гигабайт — 10⁹ байт, терабайт — 10¹² байт. Их часто используют для дисков, сетевых скоростей и маркетингового описания объёмов.

Для памяти компьютера также применяются двоичные приставки IEC: кибибайт — 1024 байта, мебибайт — 2²⁰ байт, гибибайт — 2³⁰ байт. Поэтому `1 кБ` и `1 КиБ` — не одно и то же: первый равен 1000 байт, второй — 1024 байта. Это различие важно, когда нужно точно считать объём памяти.

Пример расчёта: массив из 1000 чисел типа `int`, если один `int` занимает 4 байта, требует примерно `1000 · 4 = 4000` байт памяти без учёта служебных данных контейнера. Если нужно перевести 4096 байт в КиБ, получаем `4096 / 1024 = 4 КиБ`. Для скоростей передачи данных часто используют биты в секунду, например Мбит/с, поэтому 100 Мбит/с — это не 100 Мбайт/с. Коротко: для дисков часто используют десятичные единицы, а для оперативной памяти и адресации часто удобнее степени двойки.

## Вопрос 27. Позиционная система счисления, двоичная, восьмеричная, шестнадцатеричная

Позиционная система счисления — это система записи чисел, в которой значение цифры зависит от её позиции в числе. Например, в десятичной системе число 123 означает `1·10² + 2·10¹ + 3·10⁰`. Основание системы показывает, сколько различных цифр используется и во сколько раз «вес» позиции больше веса позиции справа.

В информатике важны двоичная, восьмеричная и шестнадцатеричная системы. Двоичная система имеет основание 2 и использует цифры `0` и `1`, поэтому напрямую соответствует битам. Например, двоичное число `1101₂` равно `1·2³ + 1·2² + 0·2¹ + 1·2⁰ = 13₁₀`. Восьмеричная система использует цифры `0–7`, а шестнадцатеричная — `0–9` и `A–F`, где `A` означает 10, `B` — 11, ..., `F` — 15; регистр букв обычно не важен, поэтому `0x2a` и `0x2A` обозначают одно значение.

Шестнадцатеричная запись удобна потому, что одна шестнадцатеричная цифра ровно соответствует четырём битам. Например, байт `11111111₂` можно записать как `FF₁₆`, а число `0x2A` равно `2·16 + 10 = 42` в десятичной системе. Поэтому hex‑запись часто используют для адресов памяти, цветов, битовых масок и содержимого бинарных файлов. В C++ одно и то же число можно записать разными способами: `13` в десятичной форме, `015` в восьмеричной, `0x0D` в шестнадцатеричной и `0b00001101` в двоичной.

Для перевода из десятичной системы в двоичную число делят на 2 и записывают остатки. Например, `25 / 2 = 12`, остаток `1`; `12 / 2 = 6`, остаток `0`; `6 / 2 = 3`, остаток `0`; `3 / 2 = 1`, остаток `1`; `1 / 2 = 0`, остаток `1`. Остатки читаются снизу вверх: `25₁₀ = 11001₂`.

Для перевода из двоичной системы в десятичную нужно разложить число по степеням двойки: `11001₂ = 1·2⁴ + 1·2³ + 0·2² + 0·2¹ + 1·2⁰ = 16 + 8 + 1 = 25₁₀`. Для перевода между двоичной и шестнадцатеричной системами двоичное число разбивают справа налево на группы по 4 бита: `101111₂ = 0010 1111₂ = 2F₁₆`. Для восьмеричной системы используют группы по 3 бита: `101111₂ = 101 111₂ = 57₈`. Типичная ошибка — читать остатки сверху вниз или забывать дополнять левую группу нулями. Коротко: основание показывает веса разрядов, а двоичная, восьмеричная и шестнадцатеричная записи — разные формы одного и того же числа.

Важно различать число и его представление. Значение 10 можно записать как `10₁₀`, `1010₂`, `12₈` или `A₁₆`; это одно и то же число, но разные способы записи. В тексте можно написать `1010₂`, чтобы показать основание системы счисления. В C++ двоичный литерал записывается как `0b1010`, шестнадцатеричный — как `0x0A`, а восьмеричный исторически записывается с начального нуля, например `012`. Это опасное место: `012` в C/C++ означает не двенадцать, а восьмеричное число, равное `10` в десятичной системе. В некоторых языках, например Python, восьмеричная запись явно помечается префиксом `0o`, например `0o12`; в стандартном C++ такой префикс для целочисленного литерала не используется. Ведущие нули в математической записи обычно не меняют значение числа, но в языках программирования могут менять основание литерала, поэтому их нужно читать по правилам конкретного языка.

Для дробной части позиционный принцип работает через отрицательные степени основания. Например, `101.11₂ = 1·2² + 0·2¹ + 1·2⁰ + 1·2⁻¹ + 1·2⁻² = 4 + 1 + 0.5 + 0.25 = 5.75₁₀`. Чтобы перевести десятичную дробную часть в другую систему, её умножают на основание и записывают целые части результатов. Например, `0.625₁₀`: `0.625·2 = 1.25`, затем `0.25·2 = 0.5`, затем `0.5·2 = 1.0`, значит `0.625₁₀ = 0.101₂`. Не всякая десятичная дробь имеет конечную двоичную запись: например, `0.1₁₀` в двоичной системе записывается бесконечной периодической дробью, поэтому в `float` и `double` хранится приближённо.

Для отрицательных целых чисел в современных компьютерах обычно используют дополнительный код. Например, чтобы получить `-5` в 8 битах, берут `5 = 00000101`, инвертируют биты `11111010` и прибавляют 1: получается `11111011`. Такой способ удобен тем, что сложение положительных и отрицательных чисел выполняется одинаковой схемой. Типичная ошибка — думать, что старший бит просто «минус», а остальные всегда обычная величина; для дополнительного кода это слишком грубое объяснение.

## Вопрос 28. Порядок байтов

Порядок байтов, или endianness, — это правило размещения байтов многобайтового значения в памяти или потоке данных. Он важен тогда, когда одно число занимает больше одного байта и его нужно записать в память, файл или передать по сети. Если число занимает несколько байтов, нужно договориться, какой байт хранится по младшему адресу: старший или младший.

В формате big‑endian первым хранится старший байт, то есть запись в памяти похожа на привычную запись числа слева направо. В формате little‑endian первым хранится младший байт. В сетевых протоколах часто используется big‑endian, а архитектуры x86 обычно используют little‑endian. Некоторые архитектуры, например ARM, могут поддерживать переключаемый порядок. Для обычного программиста порядок байтов важен при работе с бинарными файлами, сетевыми пакетами и обмене данными между разными платформами.

Например, число `0x12345678` занимает 4 байта. В big‑endian оно будет лежать как `12 34 56 78`, а в little‑endian — как `78 56 34 12`. Само значение числа не меняется, меняется только порядок байтов в памяти или файле. Для однобайтового значения порядок байтов не имеет смысла, потому что байт всего один. Типичная ошибка — путать порядок байтов с порядком битов внутри байта. Коротко: endian‑порядок отвечает на вопрос, какой байт многобайтового числа лежит первым.

Порядок байтов особенно важен при чтении бинарных файлов и сетевых пакетов. Если одна система записала число в little‑endian, а другая прочитала его как big‑endian, получится другое значение. В сетевых протоколах часто используют «сетевой порядок байтов», обычно big‑endian, чтобы разные устройства одинаково понимали числа.

## Вопрос 29. Адрес переменной, ссылка, указатель

Адрес переменной — это числовое обозначение места в памяти, где хранится объект программы. Указатель и ссылка нужны, чтобы работать не только со значением, но и с местом, где оно хранится. В C++ адрес можно получить оператором `&`. Указатель — это переменная, которая хранит адрес другого объекта или специальное значение `nullptr`, если ни на что не указывает. Например:

```cpp
int c = 0;
int* a = &c;
*a = 3;
```

Здесь `&c` берёт адрес переменной `c`, указатель `a` хранит этот адрес, а `*a` разыменовывает указатель, то есть обращается к значению по адресу. После `*a = 3` значение `c` станет равно 3. Ссылка — это псевдоним уже существующей переменной, например `int& e = c;`. После этого `e` и `c` обозначают один и тот же объект. Главное различие: указатель можно переназначать и он может быть `nullptr`, а ссылка должна быть сразу связана с объектом и обычно используется как более безопасный способ передать объект без копирования.

Адрес не следует воспринимать как обычное число для произвольных вычислений. Арифметика указателей в C++ имеет смысл только внутри массива или допустимого диапазона объекта, а обращение по неверному адресу приводит к неопределённому поведению. Поэтому указатели требуют проверки и аккуратного владения ресурсами.

Пример со ссылкой:

```cpp
void add_one(int& x)
{
    x = x + 1;
}

int value = 10;
add_one(value);
```

После вызова `value` станет равен 11, потому что функция получила ссылку на исходную переменную, а не копию. Типичные ошибки с указателями — разыменовать `nullptr`, использовать адрес локальной переменной после выхода из функции или забыть, что `*p` означает значение по адресу, а `p` — сам адрес. Коротко: указатель хранит адрес и может быть переназначен, ссылка является другим именем уже существующего объекта.

У символов `*` и `&` в C++ несколько ролей. В объявлении `int* p` звёздочка показывает, что `p` — указатель; в выражении `*p` она означает разыменование. В выражении `&c` амперсанд берёт адрес переменной, а в объявлении `int& r = c` он создаёт ссылку. На практике это место часто путают, поэтому полезно каждый раз проговаривать контекст: объявление типа или операция над уже существующим объектом.

Ещё один важный момент — стек и куча. Обычные локальные переменные функции обычно живут на стеке и уничтожаются при выходе из области видимости. Динамически выделенные объекты живут в куче и должны управляться аккуратно; в современном C++ для этого используют RAII и умные указатели, а не ручное `new`/`delete` без необходимости.
