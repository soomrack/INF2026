# Блок 3. Basic C++ syntax

## Вопрос 18. Что такое переменная?

Переменная — это именованный объект программы, предназначенный для хранения значения определённого типа. Проще говоря, это имя, через которое программа обращается к некоторому участку памяти или объекту с заданными правилами использования. У переменной есть имя, тип, область видимости, время жизни, адрес в памяти и текущее значение.

Например, в записи `int a = 10;` имя переменной — `a`, тип — `int`, а начальное значение — `10`. Тип определяет, какие значения можно хранить, сколько памяти обычно потребуется и какие операции допустимы. Значение обычной переменной может изменяться во время выполнения программы, но если переменная объявлена как `const`, её значение после инициализации менять нельзя.

Важно различать объявление, определение и инициализацию. Объявление сообщает компилятору имя и тип, определение создаёт объект, а инициализация задаёт начальное значение. В C++ лучше инициализировать переменные сразу, потому что чтение неинициализированной локальной переменной приводит к ошибочному и непредсказуемому поведению программы.

Также важно отличать переменную от её значения. Переменная — это объект с именем и временем жизни, а значение — текущее содержимое этого объекта. В выражении `a = a + 1` сначала читается старое значение `a`, затем вычисляется новое и записывается обратно в ту же переменную. Поэтому присваивание не является математическим равенством, а означает изменение состояния программы.

## Вопрос 19. Базовые типы данных

Базовые типы данных — это встроенные типы C++, которые задают вид значения, его представление в памяти и допустимые операции. Из них строятся более сложные пользовательские и библиотечные типы. К базовым относятся целочисленные типы `short`, `int`, `long`, `long long`, символьный тип `char`, логический тип `bool`, вещественные типы `float`, `double`, `long double`, а также специальный тип `void`.

Целочисленные типы могут быть знаковыми (`signed`) и беззнаковыми (`unsigned`). `bool` хранит логическое значение `true` или `false`. `char` обычно используется для хранения символа или небольшого целого значения, но сам по себе не означает «любой символ Unicode»; кроме того, будет ли обычный `char` знаковым или беззнаковым, зависит от реализации. Вещественные типы нужны для чисел с дробной частью. Размеры типов могут зависеть от платформы и компилятора, поэтому при переносимом коде нужно быть аккуратным; если нужен точный размер, используют типы вроде `std::int32_t` из `<cstdint>`.

Отдельно стоит упомянуть тип `void`. Он означает отсутствие значения: функция с типом результата `void` ничего не возвращает, а указатель `void*` может хранить адрес объекта неизвестного типа, но перед использованием такой адрес нужно привести к конкретному типу. В обычном учебном коде `void` чаще всего встречается именно в функциях, которые выполняют действие, например печать на экран, но не возвращают результат.

Важно не перепутать `signed` и `unsigned`: знаковый тип хранит отрицательные и положительные значения, а беззнаковый — только неотрицательные, зато обычно в большем положительном диапазоне при том же размере. При переполнении беззнаковое число в C++ оборачивается по модулю, например для 8 бит `255 + 1` даёт `0`. Переполнение знакового целого типа — неопределённое поведение, поэтому нельзя рассчитывать, что `INT_MAX + 1` обязательно превратится в минимальное отрицательное число. С вещественными типами тоже есть неочевидный момент: `float` и `double` хранят приближённые значения, поэтому выражение вроде `0.1 + 0.2` может быть не ровно равно `0.3`. Типичная ошибка — сравнивать вещественные числа через точное `==` там, где нужна проверка с небольшой погрешностью.

## Вопрос 20. Пользовательские типы данных

Пользовательские типы данных — это типы, созданные самим программистом на основе возможностей языка. Они нужны, чтобы описывать объекты предметной области более понятно, безопасно и выразительно, чем набором несвязанных стандартных типов.

В C++ пользовательские типы можно создавать с помощью `struct`, `class`, `enum`, а также вводить имена типов через `typedef` и `using`. Например, структура `Account` может объединять баланс, процентную ставку и методы работы со счётом. Класс позволяет не только хранить данные, но и связать с ними поведение, а перечисление (`enum` или более безопасное `enum class`) удобно для ограниченного набора значений, например режимов работы устройства.

Пользовательский тип важен не только для удобства записи. Он помогает компилятору обнаруживать ошибки: например, режим работы устройства лучше хранить как перечисление, а не как произвольное целое число, потому что допустимых режимов мало и они имеют смысловые имена.

Пример перечисления:

```cpp
enum class Mode {
    Off,
    Manual,
    Automatic
};

Mode mode = Mode::Automatic;
```

Такой тип понятнее, чем числа `0`, `1` и `2`, потому что значения имеют имена. Кроме того, `enum class` не смешивается неявно с обычными целыми числами так свободно, как старые перечисления, поэтому снижает риск случайной ошибки.

Массив не является пользовательским типом, но это важный составной тип: он хранит несколько однотипных элементов подряд, а индексация начинается с нуля. Например, в массиве `int a[3] = {10, 20, 30};` первый элемент — `a[0]`, а последний — `a[2]`. Типичная ошибка — обратиться к `a[3]`, хотя такого элемента уже нет. Если нужно хранить набор данных с переменным размером, в современном C++ обычно удобнее использовать `std::vector`.

## Вопрос 21. Циклы

Цикл — это управляющая конструкция, которая многократно выполняет блок кода, пока выполняется условие или пока не исчерпан заданный диапазон значений. Он нужен, когда одно и то же действие надо повторить несколько раз, а не писать одинаковый код вручную. В C++ основные циклы — `while`, `for` и `do while`, а также диапазонный `for` для перебора элементов контейнера.

Цикл `while` сначала проверяет условие и выполняет тело, пока оно истинно. Цикл `for` удобен, когда есть счётчик или заранее понятная схема изменения переменной. Цикл `do while` сначала выполняет тело, а потом проверяет условие, поэтому он выполнится хотя бы один раз. Диапазонный `for`, например `for (int x : values)`, удобен, когда нужно пройти по всем элементам контейнера без ручной работы с индексами. Внутри циклов можно использовать `break`, чтобы полностью выйти из цикла, и `continue`, чтобы перейти к следующей итерации.

Простой пример: посчитать сумму чисел от 1 до 5.

```cpp
int sum = 0;

for (int i = 1; i <= 5; ++i) {
    sum += i;
}
```

Здесь переменная `i` последовательно принимает значения от 1 до 5, а `sum` накапливает результат. Типичная ошибка — забыть изменить счётчик или неправильно написать условие, из-за чего цикл может стать бесконечным или выполниться на один раз больше/меньше. Коротко для устного ответа: цикл повторяет блок кода, пока выполняется условие или пока не закончится заданный диапазон значений.

## Вопрос 22. Условные операторы

Условный оператор — это управляющая конструкция, которая выбирает, какой участок кода выполнить, в зависимости от истинности логического выражения. Он даёт программе возможность принимать решения на основе входных данных и текущего состояния. Основная конструкция в C++ — `if`, которая может использоваться вместе с `else` и `else if`.

```cpp
if (a > 100) {
    k++;
} else {
    k--;
}
```

Если условие в `if` истинно, выполняется первый блок; если ложно — блок после `else`. Условные операторы позволяют программе реагировать на данные и принимать решения. Важно помнить, что условие должно иметь логический смысл, а блоки в фигурных скобках образуют отдельную область видимости. В C++ выражение в условии приводится к `bool`: ноль считается ложью, ненулевое значение — истиной, но в учебном коде лучше писать условия явно.

Например, проверка оценки:

```cpp
if (mark >= 60) {
    cout << "passed";
} else {
    cout << "failed";
}
```

Если `mark` больше или равен 60, программа выводит `passed`, иначе — `failed`. В условиях часто используют логические операции: `&&` означает «и», `||` означает «или», `!` означает отрицание. Например, `age >= 18 && has_passport` истинно только тогда, когда выполнены оба условия.

Важно не перепутать логические и побитовые операции. `&&` и `||` работают с истинностью условий, а `&` и `|` работают с отдельными битами чисел. Оператор `^` означает побитовое исключающее «или», а `~` инвертирует все биты числа; это не то же самое, что логическое отрицание `!`. Типичная ошибка — перепутать присваивание `=` и сравнение `==`: `if (x = 5)` присваивает значение, а `if (x == 5)` проверяет равенство. Для нескольких вариантов значения можно использовать `switch`, но для сложных условий с диапазонами обычно удобнее `if`. Коротко: условный оператор выбирает, какой код выполнить, в зависимости от истинности выражения.

Конструкция `switch` удобна, когда нужно выбрать действие по одному целочисленному, символьному или перечислимому значению:

```cpp
int state = 1;

switch (state) {
case 0:
    std::cout << "off";
    break;
case 1:
    std::cout << "on";
    break;
default:
    std::cout << "unknown";
    break;
}
```

В `switch` важно не забывать `break`, если не требуется намеренное «проваливание» в следующий вариант. Для проверок вида `x > 0 && x < 10` лучше подходит `if`, потому что `switch` сравнивает значение с отдельными вариантами.

## Вопрос 23. Функции

Функция — это именованный блок кода, выполняющий отдельную задачу и вызываемый из других частей программы. Она нужна, чтобы выделить действие, задать ему имя, передавать входные данные через параметры и при необходимости возвращать результат. У функции есть тип возвращаемого значения, имя, список параметров, тело и область видимости её имени.

```cpp
int sum(int a, int b)
{
    int c = a + b;
    return c;
}
```

В этом примере функция `sum` принимает два параметра типа `int`, складывает их и возвращает результат. Функции нужны, чтобы не повторять один и тот же код, делить программу на понятные части и скрывать детали реализации. Если функция ничего не возвращает, её типом указывают `void`. Объявление функции сообщает компилятору её интерфейс, а определение содержит тело; в больших программах объявления часто помещают в заголовочные файлы.

Функцию удобно понимать как схему «входные данные → действие → результат». У `sum` входные данные — `a` и `b`, действие — сложение, результат — возвращаемое число. Типичные ошибки: забыть `return` в функции с не-`void` типом, перепутать параметры и локальные переменные, пытаться использовать локальную переменную за пределами функции. Коротко: функция — это именованный фрагмент программы, который можно вызывать много раз с разными аргументами.

Параметры можно передавать по значению, по ссылке и через указатель. При передаче по значению функция получает копию, поэтому исходная переменная не меняется. При передаче по ссылке, например `void inc(int& x)`, функция работает с исходной переменной. Если менять объект не нужно, но копировать его дорого, используют константную ссылку, например `const std::string&`. При передаче через указатель, например `void inc(int* p)`, нужно передать адрес и проверить, что указатель не равен `nullptr`. Важно объяснять это на примере: если функция должна изменить исходный объект, одной передачи по значению недостаточно.

Ещё одно важное понятие — сигнатура функции. Обычно под ней понимают имя функции и типы её параметров. В C++ можно перегружать функции: иметь несколько функций с одним именем, но разными параметрами. Например, `print(int)` и `print(std::string)` — разные перегрузки. Тип возвращаемого значения сам по себе не позволяет различить перегрузки, поэтому две функции с одинаковыми параметрами и разным возвращаемым типом объявить нельзя.

## Вопрос 24. Область видимости

Область видимости — это часть программы, где имя переменной, функции, типа или другого объекта доступно для обращения. В C++ локальная область видимости обычно ограничивается ближайшими фигурными скобками, внутри которых объект объявлен.

```cpp
int main()
{
    int a = 0;

    for (int k = 0; k < 10; ++k) {
        int b = 1;
        a += b;
    }

    return 0;
}
```

В этом примере переменная `a` доступна почти во всей функции `main`, переменная `k` доступна внутри цикла `for`, а переменная `b` — только внутри тела цикла. После выхода из области видимости локальная переменная уничтожается, и обращаться к ней нельзя.

Практический смысл области видимости — не давать именам мешать друг другу и ограничивать время жизни временных данных. Например, счётчик `k` нужен только внутри цикла, поэтому его удобно объявлять прямо в `for`. Нужно различать область видимости и время жизни: имя может быть недоступно в некотором месте программы, хотя объект ещё существует, и наоборот, после уничтожения объекта старый адрес уже нельзя безопасно использовать. Типичная ошибка — объявить переменную внутри блока, а потом попытаться использовать её после закрывающей фигурной скобки. Коротко: область видимости отвечает на вопрос, где имя переменной или функции доступно в программе.

В C++ есть несколько уровней области видимости: блочная область внутри `{}`, область функции, область класса, пространство имён и глобальная область. Если во внутренней области объявить имя, совпадающее с внешним, оно скрывает внешнее имя. Такой приём возможен, но в учебном и промышленном коде им лучше не злоупотреблять, потому что он ухудшает читаемость и может приводить к ошибкам.
